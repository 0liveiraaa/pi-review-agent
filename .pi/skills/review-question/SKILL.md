# 出题技能

## 零、难度分级

题目难度由「知识点广度 × 认知层次」两个维度交叉定义，共 5 个级别:

| 级别 | 广度 | 认知 | 含义 | 适用题型 |
|------|------|------|------|----------|
| **S-R** | Single (单一概念) | Recall (记忆/识别) | 考察对单个知识点的记忆。题目直接，不设陷阱。 | 正误判断 |
| **S-U** | Single (单一概念) | Understand (理解/区分) | 考察对单个知识点的深入理解。包含一个需要思考的混淆点。 | 正误判断、单项选择 |
| **M-U** | Multi (2-3个关联概念) | Understand (理解/区分) | 考察多个关联知识点的联动理解。选项/陈述涉及不同但相关的概念。 | 单项选择 |
| **M-A** | Multi (2-3个关联概念) | Analyze (分析/评价) | 考察多概念综合分析能力。题目需要推理、对比或场景分析。 | 单项选择、简述题 |
| **C-A** | Chain (知识链条) | Analyze (分析/评价) | 跨知识点综合考察。题目需要串联多个概念形成完整的分析链。 | 简述题 |

### 出题时按难度级别控制:

- **S-R**: 直接问定义/规则。例: "拷贝构造函数在何时被调用？"
- **S-U**: 考概念区分。例: "判断: 拷贝构造函数和赋值运算符的触发时机相同。"
- **M-U**: 关联 2 个概念比较。例: "关于浅拷贝和深拷贝的区别，以下正确的是？"
- **M-A**: 多概念综合推理。例: "如果类没有定义拷贝构造函数，以下场景会出现什么问题？"
- **C-A**: 知识链条。例: "从拷贝构造到赋值函数，阐述对象复制的完整策略。"

### 核心原则:

- **广度**决定题目涉及几个知识点，**认知**决定思考深度
- 干扰项应与题目同难度级别: S 级干扰项来自同一知识点的误区，M 级干扰项来自关联知识点的混淆

## 一、题型模板

### 1.1 正误判断题 (judgment)

```
【题型】正误判断
【知识点】{知识点名称}
【难度】{S-R / S-U}

判断以下说法是否正确:

"{一句陈述，包含一个容易混淆的细节}"

请回答「正确」或「错误」。
```

**生成规则:**
- 陈述简短明确，不超过两句话
- 考察概念区分（如拷贝构造 vs 赋值的触发时机）
- 错误不要设计得太明显，让用户需要思考

### 1.2 单项选择题 (choice)

```
【题型】单项选择题
【知识点】{知识点名称}
【难度】{S-U / M-U / M-A}

{题目描述}

A. {选项A}
B. {选项B}
C. {选项C}
D. {选项D}

请选择你认为正确的选项 (A/B/C/D)。
```

**生成规则:**
- 4 个选项，1 个正确，3 个干扰
- 干扰项来自常见误区
- 不出现「以上都对/都错」类选项
- 选项互斥且有迷惑性

### 1.3 简述题 (short_answer)

```
【题型】简述题
【知识点】{知识点名称}
【难度】{M-A / C-A}

{开放式问题}

请用你的话回答，覆盖关键要点即可。
```

**生成规则:**
- 问题让用户组织语言，非简单回忆
- 常见类型: 对比辨析、原理解释、场景分析
- 判题按要点给分，不要求逐字匹配
