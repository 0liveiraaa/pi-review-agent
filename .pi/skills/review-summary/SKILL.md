# 复盘总结技能

## 一、每题复盘格式

每道题结束后，生成一条复盘 JSON。用 ```json 包裹。

```json
{
  "section": "小节编号和标题",
  "question_summary": "题目的一句话概括",
  "user_answer": "用户的回答",
  "is_correct": true,
  "core_learning": "本题的核心知识点/收获",
  "weak_point": "暴露的薄弱点（正确则为null）",
  "error_root_cause": "错误根因（正确则为null）",
  "knowledge_chain": ["知识点1", "知识点2"]
}
```

**复盘要点:**
- `question_summary`: 不要照抄题目全文，用一句话概括题目在考什么
- `core_learning`: 本题最关键的收获是什么
- `weak_point`: 如果答错，具体是哪个概念混淆或遗漏了
- `error_root_cause`: 错误根因而非表象。例: "混淆了栈区和堆区的管理方式" 而非 "选错了B"
- `knowledge_chain`: 用 2-3 个知识点串联本题涉及的知识链

## 二、会话总结报告模板

会话结束时，根据所有复盘记录生成一份总结报告。输出为 MD 格式。

### 报告结构

```markdown
# 📊 复习会话总结

**日期**: {日期}
**范围**: {复习范围}
**成绩**: {正确数}/{总数} ({正确率}%)

---

## 一、总体评价

（2-3句话概括本次会话表现）

## 二、逐题回顾

| # | 小节 | 结果 | 核心收获 |
|---|------|------|----------|
| 1 | 11.1 存储空间 | ✅ | 栈区自动管理，堆区手动管理 |
| 2 | 11.2 静态存储 | ❌ | 混淆了static局部变量的存储位置 |
| ... | ... | ... | ... |

## 三、薄弱环节

- **{薄弱点1}**: 具体表现 + 建议
- **{薄弱点2}**: ...

## 四、知识体系

用 → 箭头串联本次复习建立的知识关联

## 五、下次复习建议

1. **优先回顾**: {最需要强化的知识点}
2. **建议范围**: {推荐的下次复习范围}
3. **难度建议**: {基于本次表现推荐的起始难度}
```

### 报告要点

- **逐题回顾**表格: 每题一行，核心收获用一句话概括
- **薄弱环节**: 从错题中提取模式，不是逐题罗列
- **知识体系**: 优先用 `→` 箭头串知识链
- **下次建议**: 具体到知识点名称和难度级别
