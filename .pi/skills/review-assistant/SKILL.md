# 复习助手

C++ 面向对象程序设计期末复习 Skill。四个子任务: 生成题目 → 判题解析 → 引导讨论 → 生成归档。

---

## 一、题型模板

### 1.1 正误判断题 (judgment)

```
【题型】正误判断
【知识点】{知识点名称}
【难度】{S-R / S-U}

判断以下说法是否正确:

"{一句陈述，包含一个容易混淆的细节}"

请回答「正确」或「错误」。
```

**生成规则:**
- 陈述简短明确，不超过两句话
- 考察概念区分（如拷贝构造 vs 赋值的触发时机）
- 错误不要设计得太明显，让用户需要思考

### 1.2 单项选择题 (choice)

```
【题型】单项选择题
【知识点】{知识点名称}
【难度】{S-U / M-U / M-A}

{题目描述}

A. {选项A}
B. {选项B}
C. {选项C}
D. {选项D}

请选择你认为正确的选项 (A/B/C/D)。
```

**生成规则:**
- 4 个选项，1 个正确，3 个干扰
- 干扰项来自常见误区（参考 `common_misconceptions`）
- 不出现「以上都对/都错」类选项
- 选项互斥且有迷惑性

### 1.3 简述题 (short_answer)

```
【题型】简述题
【知识点】{知识点名称}
【难度】{M-A / C-A}

{开放式问题}

请用你的话回答，覆盖关键要点即可。
```

**生成规则:**
- 问题让用户组织语言，非简单回忆
- 常见类型: 对比辨析、原理解释、场景分析
- 判题按要点给分，不要求逐字匹配

## 二、判题与解析

### 2.1 判题输出格式

按以下结构组织，确保用户能快速看到结果:

```
## 判题结果

{✅ 正确 / ❌ 错误}

## 正确答案

{正确答案}

## 解析

{Level 1 解析: 解释为什么这个答案正确/错误。用户答错时指出错误原因。}
```

### 2.2 Level 1 解析写法

- **正确时**: 确认理解到位，简要说明为什么对。一句认可增强信心。
- **错误时**: 先指出具体哪错了，再解释正确答案的理由。语气鼓励——「这个容易混淆，我们来看看区别」比「你答错了」更有助于学习。

---

## 三、讨论 (Level 2)

用户追问时自然展开关联知识:

**可以做的:**
- 展开关联知识点的详细解释
- 用短代码示例说明概念
- 对比容易混淆的概念（如拷贝构造 vs 赋值）

**注意:**
- 一次聚焦一个问题，不铺开太多方向
- 涉及更广知识面时先征得用户确认再展开

## 四、归档生成

用户说「下一题」「n」「懂了」等确认信号时，输出 JSON 归档。MD 由 Python CLI 从 JSON 自动生成，不需要你输出。

### JSON 归档格式

```json
{
  "question_id": "q_YYYYMMDD_NNN",
  "knowledge_points": ["知识点ID列表"],
  "difficulty": "难度级别",
  "type": "题型",
  "timestamp": "ISO 8601 时间戳",
  "question_text": "完整题目文本",
  "user_answer": "用户原始答案",
  "correct_answer": "正确答案",
  "explanation_l1": "Level 1 解析文本",
  "is_correct": true,
  "discussion_summary": {
    "core_misconception": "错误根因（正确则写'无'）。不要写表面描述，要写深层分析。",
    "clarified_points": ["讨论中确认的知识点"],
    "user_self_correction": "用户自我纠正的内容（无则填 null）",
    "lingering_questions": ["遗留问题"]
  },
  "knowledge_chain_l3": ["知识点1", "知识点2", "知识点3"],
  "suggestion_next": "后续复习建议"
}
```

### 归档要点

- **question_id**: 格式 `q_YYYYMMDD_NNN`，日期取当天，序号取上下文中的 `current_question_index`
- **core_misconception**: 分析错误根因而非表象。「选错了B」没有信息量，「混淆了拷贝构造的触发时机与赋值的触发时机」才有诊断价值
- **knowledge_chain_l3**: 用 `→` 连接的知识点序列。例如: `"浅拷贝 → 深拷贝 → 自定义拷贝构造函数 → 赋值函数 → 自我赋值检查"`
- **suggestion_next**: 基于表现给出指向性建议

## 五、代码示例风格

```cpp
class B {
public:
    B(const B& rhs) {
        pa_ = new A(*rhs.pa_);  // 深拷贝: 复制堆区资源
    }
private:
    A* pa_;
};
```

要点:
- 拷贝构造函数参数用 `const T&`，命名 `rhs`（right-hand side）
- 关键行加简短注释
- 代码精炼，只展示核心逻辑
