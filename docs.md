## 文档结构（六大块）

**1. 一句话开场 + 解决什么问题**

* 用一句话点题：把你的笔记变成一个「私人 AI 助教」，它会出题、判题、讲解、盯进度。
* 对比痛点：传统复习 = 被动重读笔记；本工具 = 主动回忆 + 个性化追踪。
* 点明它不是一个聊天机器人，也不是网课平台（呼应「产品定位」文档）。

**2. 核心功能（重点用类比）**

* 把笔记变成「复习资料包」(profile)：8 步流水线，用人话讲（像助教帮你把零散笔记整理成考点手册、卡片、章节笔记）。
* 三种复习模式：卡片+练习 / 直接练习 / 章节学习——用「背概念 / 考前刷题 / 系统学」三个场景讲。
* 五级难度：知识广度 × 认知深度，配 1 句话示例（S-R 识别 → C-A 综合设计）。
* 题型：判断/单选/多选/简答，以及「按科目适配」（数学用推导、编程用代码、文科用论述）。
* 多层复盘：L1 即时解析 / L2 追问深入 / L3 全景梳理——类比「对答案 → 追问为什么 → 串起整条知识线」。
* 智能追踪：错题本（自动优先复现）、知识链路（掌握度衰减+薄弱预警）、学习画像（长期记住你哪里弱）。

**3. 实现方式（类比 + 关键概念）** 用一个贯穿全文的比喻收束整段：**「代码像一位严谨的舞台监督，AI 像一位临场发挥的演员。」**

* **代码负责的（舞台监督）** ：渲染卡片/题目界面、存档、状态、路径安全、格式校验——保证每次都一样、不会出错。
* **AI 负责的（演员）** ：理解笔记、出题、判题、讲解、追问——负责所有「需要动脑理解」的部分。
* **9 个工具** ：用「AI 只能通过几个固定按钮（工具）和系统对话」来解释——每个按钮要填的内容都有严格格式（schema），AI 不能乱来。举 2-3 个工具名即可（如 `review_answer` 出题、`review_archive` 存档）。
* **资料包生命周期** ：draft（草稿）→ active（启用）→ archived（归档可回滚），类比「先打草稿 → 确认无误再启用 → 旧版自动存档以防万一」。
* **数据落点** ：用大白话说清楚你的答题、错题、学习画像存在哪里、属于谁（私有 `_user/` 不随资料包分享）。

**4. 设计思路（讲为什么）**

* **「代码稳定，Agent 灵活」** 是全项目核心哲学（来自文档原话），解释为什么这样切分。
* **资料包是核心产品资产** ，不是临时缓存——可迁移、可分享、可版本化。
* **结构化 JSON + schema 校验** ：为什么不让 AI 自由发挥（不稳定），而是逼它填固定格式（可控）。
* **质量前置、诚实标注** ：资料低置信度要标 `<!-- UNCERTAIN -->`，质量报告阻断「带病启用」——呼应「不让坏资料放大成坏体验」。
* **demo 是销售入口** ，不是样例代码：1 分钟内看懂「卡片、考点、章节、答题、总结」的关系。

**5. 后续演进（按路线图分阶段讲）** 基于 `产品迭代路线.md`、`后续演进指导.md`、`通用知识Profile框架设想.md`，提炼成普通人能懂的方向：

* **近期** ：把安装/首次体验做稳（doctor 诊断）、让笔记生成资料包更可靠（质量报告 + 按质量报告逐项修订）、资料包可导入导出（同学间分享，不用各自重新生成）。
* **中期** ：「今日推荐」复习队列（系统告诉你今天该复习什么）、遗忘曲线策略、学科适配层（不同科目有不同题型和判题标准）。
* **长期愿景** ：从「复习插件」升级成「个人知识学习运行时」——同一套框架支持课程复习、论文阅读、读书笔记、真题训练、技术文档、语言学习（用「换不同的资料包，就像换不同的课本」来类比 profile_type）。
* **暂缓事项** （诚实说明）：PDF/Word 原生导入、Web UI、云同步、多人班级看板——解释为什么先不做（先把核心做稳）。

**6. 一句话收尾**

* 回到「短期把安装和 demo 做稳，中期把资料包生产和分享做强，长期把推荐和学科适配做成壁垒」这句来自文档的路线总结。
