## 重点难点

### 重点

1. **多态的定义与分类**：考察方式为定义判断题（选择题或判断题）。核心考察点：区分"同名不同体"与"同名同体"。备考建议：熟记多态的两种分类（静态/动态）及其典型机制。

2. **C++虚机制分派规则**：本章最高频考点。考察方式为给定代码分析题（选择、简答）。核心考察点：目标对象按动态类型、参数按静态类型。备考建议：死记"目标动态、参数静态"八字，然后在代码题中逐层应用。

### 难点

1. **目标对象 vs 参数的分派差异**：最容易混淆的点。考生容易认为参数也按动态类型分派。区分关键：目标对象的分派是运行期通过虚表进行的；参数的重载匹配是编译期根据声明类型完成的。避坑方法：每次遇到 `pa->f(arg)`，先问"pa 的动态类型是什么？"再问"arg 的静态类型是什么？"

2. **静态多态是否算"真正"多态**：概念理解题中易出错。区分关键：多态的定义是"同名不同体"，函数重载和模板也满足这个条件，区别仅在于决定阶段是编译期。避坑方法：不要把"多态"等同于"虚函数"。

## 典型例题

### 例 1（选择，S-U 难度）

以下哪项不属于 C++ 中的多态？

A. 函数重载
B. 模板实例化
C. 虚函数动态绑定
D. 内联函数展开

**正确答案**：D

**解析**：
- A ✅ 函数重载：同名不同参数表，编译期选择不同函数体 → 静态多态
- B ✅ 模板：`My<int>::f` 和 `My<float>::f` 是不同的函数体 → 静态多态
- C ✅ 虚函数动态绑定：根据动态类型运行期确定函数体 → 动态多态
- D ❌ 内联函数展开：只是编译器将函数体插入调用点，不产生"同名不同体"的效果，并非多态

### 例 2（判断，M-U 难度）

```cpp
class A {
public:
    virtual void f(A*) { cout << "A::f(A*)"; }
    virtual void f(void*) { cout << "A::f(void*)"; }
};
class B : public A {
public:
    void f(A*) override { cout << "B::f(A*)"; }
    void f(void*) override { cout << "B::f(void*)"; }
};

int main() {
    A* pa = new B;
    void* pv = nullptr;
    pa->f(pv);
}
```

上述代码输出为 `B::f(void*)`。

**正确答案**：✅ 正确

**解析**：
- 目标对象 `pa` 动态类型为 `B*` → 从 B 类开始查找虚函数
- 参数 `pv` 静态类型为 `void*` → 匹配 `f(void*)` 版本
- 两者结合 → `B::f(void*)`

## 复习建议

1. 优先掌握：虚机制分派规则（目标动态、参数静态）、多态分类（静态 vs 动态）
2. 易错关注：不要混淆参数分派和动态绑定、不要认为静态多态不是多态
3. 推荐用时：1-2 小时（概念理解 + 代码分析练习）
