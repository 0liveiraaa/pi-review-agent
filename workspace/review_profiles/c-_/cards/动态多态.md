# 动态多态

> **ID**: dynamic_polymorphism | **难度**: S-U | **标签**: 多态、动态多态、虚函数

## 定义

动态多态是指调用目标至少部分依赖运行时信息。在 C++ 中，动态多态主要通过虚函数和虚机制实现——由目标对象的动态类型决定最终调用的函数体。

## 关键要点

1. **虚函数是关键**：基类中用 `virtual` 声明的函数，派生类用 `override` 重写。
2. **动态类型决定行为**：通过基类指针/引用调用虚函数时，实际执行的是其动态类型（真实类型）对应的版本。
3. **分派两分**：目标对象按动态类型分派，但参数重载匹配仍按静态类型进行（详见"虚机制分派规则"）。

## 代码示例

```cpp
#include <iostream>
using namespace std;

class Base {
public:
    virtual ~Base() = default;
    virtual void greet() {
        cout << "Hello from Base" << endl;
    }
};

class Derived : public Base {
public:
    void greet() override {
        cout << "Hello from Derived" << endl;
    }
};

int main() {
    Base* p = new Derived;
    p->greet();   // 输出: Hello from Derived（动态分派）
    delete p;
    return 0;
}
```

## 常见误区

| # | ❌ 误区 | ✅ 正确理解 |
|---|---------|------------|
| 1 | 动态多态中所有参数也按动态类型分派 | 只对目标对象如此，参数重载匹配仍按静态类型 |
| 2 | 虚函数一定很慢 | 现代 C++ 中虚函数开销很小，只在指针引用调用时产生间接跳转 |

## 关联

- **静态多态**：编译期多态，与动态多态对应互补
- **C++虚机制分派规则**：深入理解目标对象与参数的分派差异

## 出题提示

通过基类指针指向派生类对象后的虚函数调用，考察考生是否理解动态多态的基本表现；设问"为什么一行 p->func() 可能不知道结果"。
