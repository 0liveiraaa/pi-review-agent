# 子类型化与面向接口编程

> **ID**: subtype_interface_programming | **难度**: M-A | **标签**: 设计、子类型化、面向接口

## 定义

子类型化（subtype polymorphism）指派生类对象可以被视为基类类型的子类型，在基类指针/引用处使用。面向接口编程则是指客户代码只依赖稳定接口（基类/抽象类），不依赖具体实现类，从而获得扩展性和可维护性。

## 关键要点

1. **子类型 ≠ 继承**：继承是代码复用机制，子类型是类型语义关系。不是所有继承都构成子类型关系（如私有继承不是子类型）。
2. **面向接口的核心**：定义抽象基类（或纯虚类），客户代码通过基类指针/引用操作，不感知具体类型。
3. **扩展方式**：新增功能 = 新增子类，而不是修改已有客户代码。

## 代码示例

```cpp
#include <iostream>
#include <vector>
using namespace std;

// 抽象接口
class SortStrategy {
public:
    virtual ~SortStrategy() = default;
    virtual void sort(vector<int>& data) const = 0;
};

// 具体策略 1
class QuickSort : public SortStrategy {
public:
    void sort(vector<int>& data) const override {
        cout << "Using QuickSort" << endl;
        // 快速排序实现...
    }
};

// 具体策略 2
class MergeSort : public SortStrategy {
public:
    void sort(vector<int>& data) const override {
        cout << "Using MergeSort" << endl;
        // 归并排序实现...
    }
};

// 客户代码——面向 SortStrategy 接口
void processData(vector<int>& data, const SortStrategy& strategy) {
    strategy.sort(data);
}

int main() {
    vector<int> data = {3, 1, 4, 1, 5};
    QuickSort qs;
    MergeSort ms;
    
    processData(data, qs);  // Using QuickSort
    processData(data, ms);  // Using MergeSort
    
    // 新增 HeapSort，processData 无需修改
    return 0;
}
```

## 常见误区

| # | ❌ 误区 | ✅ 正确理解 |
|---|---------|------------|
| 1 | 面向接口编程只是 Java 的概念 | C++ 的抽象基类和纯虚函数完全支持 |
| 2 | 所有继承都构成子类型关系 | 私有继承、部分实现继承不一定构成子类型 |
| 3 | 子类型化只能通过虚函数实现 | 模板的静态多态也能达到类似效果（但类型擦除方式不同） |

## 关联

- **虚机制的设计价值**：子类型化是虚机制设计价值的具体体现形式
- **动态多态**：虚函数是实现子类型多态的 C++ 语法机制

## 出题提示

通过设计场景（如 Shape/绘图、Strategy 模式）考察选择——哪些情况适合使用虚机制实现扩展性；判断违反 OCP 的设计（如用 enum + switch 替代虚函数）。
