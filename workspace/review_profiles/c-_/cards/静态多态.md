# 静态多态

> **ID**: static_polymorphism | **难度**: S-U | **标签**: 多态、静态多态、编译期

## 定义

静态多态是指根据目标对象的静态类型和参数表的静态类型，在编译期确定调用目标。发生阶段为**编译期**，不产生运行期分派开销。C++ 中的典型形式包括函数重载和模板。

## 关键要点

1. **函数重载**：同一作用域内函数名相同、参数表不同的多个版本。编译器根据调用时参数的静态类型选择匹配版本。
2. **模板**：模板实例化后不同特化类型产生不同类，同名成员函数属于不同类中的不同函数体。
3. **静态多态也是多态**：满足"同名不同体"的核心定义，区别仅在于决定阶段是编译期而非运行期。

## 代码示例

```cpp
#include <iostream>
using namespace std;

// --- 函数重载示例 ---
class Printer {
public:
    void show(int i) {
        cout << "int: " << i << endl;
    }
    void show(double d) {
        cout << "double: " << d << endl;
    }
};

// --- 模板示例 ---
template <typename T>
class Wrapper {
public:
    void identify(T val) {
        cout << typeid(T).name() << endl;
    }
};

int main() {
    Printer p;
    p.show(42);       // int: 42（编译期选 show(int)）
    p.show(3.14);     // double: 3.14（编译期选 show(double)）

    Wrapper<int> w1;
    Wrapper<double> w2;
    w1.identify(1);   // int
    w2.identify(1.0); // double
    return 0;
}
```

## 常见误区

| # | ❌ 误区 | ✅ 正确理解 |
|---|---------|------------|
| 1 | 重载不算多态，只是语法糖 | 它符合"同名不同体"定义，是编译期多态 |
| 2 | 所有多态都发生在运行期 | 静态多态发生在编译期，没有运行期开销 |

## 关联

- **多态的含义**：静态多态是广义多态的一种
- **动态多态**：运行期多态，与静态多态对应互补

## 出题提示

提供函数重载或模板代码，判断哪些调用在编译期已确定分派；注意陷阱——让考生误以为所有"同名不同体"都是虚函数。
