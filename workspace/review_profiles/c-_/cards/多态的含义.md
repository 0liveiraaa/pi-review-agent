# 多态的含义

> **ID**: polymorphism_concept | **难度**: S-U | **标签**: 多态、基础概念

## 定义

多态性（polymorphism）指：相同的消息请求，在不同对象或不同参数条件下，执行不同的代码体，从而产生不同结果。在 C++ 中常见形式为 `object.f(args)`——消息名同样是 `f`，但最终执行的函数体可能不同。

## 关键要点

1. **多态是广义概念**：不限于虚函数。函数重载、模板、虚函数都是多态的不同实现方式。
2. **判断标准**："同名不同体"是核心。同一函数调用表达式在不同上下文中产生不同行为。
3. **与 OOP 的关系**：仅靠抽象、封装、继承只能做到"基于对象"；加入多态才更完整地体现面向对象。

## 代码示例

```cpp
// 以下三个场景都体现了多态性

// 场景 1：函数重载（静态多态）
void print(int i) { cout << i; }
void print(double d) { cout << d; }
// print(10) 和 print(3.14) 执行不同函数体

// 场景 2：模板（静态多态）
// My<int>::f 和 My<float>::f 是不同的函数体

// 场景 3：虚函数（动态多态）
A* p = new B;
p->func();  // 调用 B::func()，而非 A::func()
```

## 常见误区

| # | ❌ 误区 | ✅ 正确理解 |
|---|---------|------------|
| 1 | 多态只指虚函数动态绑定 | 静态多态（重载、模板）也是多态 |
| 2 | 函数重载和模板不算"真正"多态 | 它们符合"同名不同体"的定义，属于编译期多态 |

## 关联

- **静态多态**：编译期决定的同名不同体
- **动态多态**：运行期参与决定的同名不同体

## 出题提示

考察定义型理解：给出多个代码片段让考生判断哪些体现了多态；注意陷阱——只有同名不同体才算，同名同体不算。
