# C++虚机制分派规则

> **ID**: virtual_dispatch_rule | **难度**: M-U | **标签**: 虚机制、分派、动态绑定

## 定义

C++ 虚函数分派中，目标对象（调用者）按**动态类型**决定虚函数到哪个类的版本，而参数表达式（如果有重载）按**静态类型**匹配重载版本。这两个决策分别在运行期和编译期发生。

## 关键要点

1. **目标对象 → 动态类型**：`pa->func()` 中，`pa` 的动态类型（实际指向的对象的类型）决定在哪个类中查找 `func`。
2. **参数 → 静态类型**：如果 `func` 有重载版本，参数的声明类型（静态类型）决定匹配哪个重载，即使参数实际指向派生类对象。
3. **两部分分离**：虚函数分派只解决目标对象的动态绑定；参数的重载匹配是编译期独立完成的。

## 代码示例

```cpp
#include <iostream>
using namespace std;

class A {
public:
    virtual ~A() = default;
    virtual void f(A*)   { cout << "A::f(A*)" << endl; }
    virtual void f(void*){ cout << "A::f(void*)" << endl; }
};

class B : public A {
public:
    void f(A*) override   { cout << "B::f(A*)" << endl; }
    void f(void*) override { cout << "B::f(void*)" << endl; }
};

int main() {
    A* pa = new B;   // pa 静态类型 A*，动态类型 B*
    B b;
    A* p_to_b = &b;  // 静态类型 A*（指向 B 对象）

    pa->f(p_to_b);
    // 分派过程：
    // step 1: 目标 pa 动态类型 = B → 从 B 开始查找 f
    // step 2: 参数 p_to_b 静态类型 = A* → 匹配 f(A*)
    // 结果: B::f(A*)
    
    delete pa;
    return 0;
}
```

## 常见误区

| # | ❌ 误区 | ✅ 正确理解 |
|---|---------|------------|
| 1 | 参数也按动态类型匹配 | 参数重载匹配是编译期行为，只看静态类型 |
| 2 | 认为 `p->func()` 只看 p 的静态类型 | 虚函数调用看 p 的动态类型 |
| 3 | 混淆虚函数分派和重载 | 虚分派解决目标对象动态绑定，重载在编译期独立完成 |

## 关联

- **动态多态**：虚机制分派是动态多态的具体实现规则
- **静态多态**：对比理解——静态多态全在编译期决定

## 出题提示

多层继承结构中，给定 `pa->f(arg)` 场景，arg 的静态类型和实际类型不同时，判断最终调用哪个函数。这是高频考点和常见混淆点。
