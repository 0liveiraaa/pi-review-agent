# 19.1 静态多态与动态多态

## 核心概念

- **多态性**：相同的消息请求，在不同对象或不同参数条件下，执行不同的代码体，从而产生不同结果。
- **静态多态**：根据目标对象的静态类型和参数表的静态类型，在编译期确定调用目标。
- **动态多态**：调用目标至少部分依赖运行时信息，C++ 中主要通过虚函数和虚机制实现。

## 知识点详解

### 多态的含义

多态是面向对象程序设计的核心之一。仅有抽象、封装和继承，更接近"基于对象"的程序设计；加入多态后才更完整地体现面向对象。

在 C++ 中，常见表达形式是 `object.f(args)`——消息名同样是 `f`，但由于目标对象或参数不同，最终执行的函数体可能不同。

### 静态多态

静态多态的判定完全在编译期完成。C++ 中两种典型形式：

1. **函数重载**：同一作用域内，函数名相同、参数表不同的多个版本。编译器根据调用时参数的静态类型选择匹配的版本。

2. **模板**：模板实例化后，`My<int>` 和 `My<float>` 是不同的类，虽然都写作 `f`，但属于不同类中的不同函数体。

**关键点**：静态多态也是在"同名不同体"的意义上属于多态。它是编译期决定的，不产生运行期开销。

### 动态多态

C++ 的动态多态主要通过虚函数实现。

**分派规则（核心考点）**：
- 目标对象 → 看**动态类型**（运行时决定）
- 参数列表 → 看**静态类型**（编译期决定）

```cpp
A* pa = new B;
B b;
pa->f(&b);   // 目标对象 pa 的动态类型决定到哪个类
             // 参数 &b 的静态类型 A* 决定匹配哪个重载版本
```

这意味着：如果参数是基类指针，即使它实际指向派生类对象，重载匹配仍按基类指针的静态类型进行。

### C++ 动态多态的边界

| 决定因素 | C++ 虚机制中的处理 |
|---|---|
| 目标对象 | 看动态类型 |
| 参数列表 | 看静态类型 |

虚函数分派解决的是**目标对象的动态绑定**；参数重载匹配仍发生在编译期。

## 关键示例

### 函数重载（静态多态）

```cpp
#include <iostream>
using namespace std;

class A {
public:
    void f(int) {
        cout << "int" << endl;
    }
    void f(char*) {
        cout << "char*" << endl;
    }
};

int main() {
    A obj;
    obj.f(10);      // 输出: int
    obj.f("abc");   // 输出: char*
    return 0;
}
```

### 模板（静态多态）

```cpp
#include <iostream>
using namespace std;

template <typename T>
class My {
public:
    void f(T value) {
        cout << typeid(T).name() << endl;
    }
};

int main() {
    My<int> obj1;
    My<float> obj2;
    obj1.f(2);      // 输出: int
    obj2.f(2.0f);   // 输出: float
    return 0;
}
```

### 动态多态分派示例

```cpp
#include <iostream>
using namespace std;

class A {
public:
    virtual ~A() = default;
    virtual void f(A*) { cout << "A::f(A*)" << endl; }
    virtual void f(class B*) { cout << "A::f(B*)" << endl; }
};

class B : public A {
public:
    void f(A*) override { cout << "B::f(A*)" << endl; }
    void f(B*) override { cout << "B::f(B*)" << endl; }
};

int main() {
    A* pa = new B;
    B b;
    A* pa_to_b = &b;
    
    pa->f(pa_to_b);  // 目标 pa 动态类型 B -> B::f(A*)
                      // 参数 pa_to_b 静态类型 A* -> 匹配 f(A*) 版本
    // 输出: B::f(A*)
    
    delete pa;
    return 0;
}
```

## 常见误区

- ❌ 认为只有虚函数才算多态 → ✅ 函数重载和模板也是多态（静态多态），它们在编译期完成分派
- ❌ 认为动态多态中参数也按动态类型分派 → ✅ C++ 中虚机制的目标对象按动态类型、参数按静态类型
- ❌ 混淆"多态"和"虚函数"，以为它们是同一概念 → ✅ 多态是更广义的概念，虚函数是 C++ 实现动态多态的一种机制
