# 题目归档: q_20260605_003

## 题目
## 11.9 其它说明 — 内容简述

本节补充了几个动态内存管理中的重要细节：

| 知识点 | 核心要点 |
|--------|----------|
| **`delete void*`** | 丢失类型信息，编译器无法知道该调哪个析构函数，对象资源可能泄漏 |
| **`delete` 不置空指针** | `delete p;` 后 `p` 仍是原地址，变成**悬空指针**；建议手动 `p = nullptr;` |
| **内存泄漏原因** | `new`/`delete` 分散在不同路径，异常或提前返回导致未释放 |
| **写时复制（COW）** | 读时共享，写时复制——引用计数 > 1 时先复制再修改，避免影响其他共享者 |
| **定位分配（placement new）** | 在已有内存上构造对象，不分配新空间；销毁时**手动调用析构函数**，不能用 `delete` |

## 解析

这个点确实容易混淆，我们来看区别：

**placement new 和普通 `new` 的本质区别：**

| 对比点 | 普通 `new` | placement new |
|--------|-----------|---------------|
| 分配内存 | ✅ 自动在堆区分配 | ❌ 使用已有内存 |
| 构造对象 | ✅ 在分配的内存上构造 | ✅ 在指定内存上构造 |
| 释放方式 | `delete p`（析构 + 释放） | **只能手动调析构**，不能 `delete` |

```cpp
char data[sizeof(A)];

A* p = new (data) A(10);   // placement new，使用栈上 data 的内存

// ❌ 错误：delete p; 会试图释放 data 的内存（栈内存不可释放！）
// ✅ 正确：
p->~A();                    // 手动调用析构函数，清理资源
```

使用 `delete p;` 的问题：
1. `delete` 会调用 `operator delete(p)` 释放内存——但 placement new 用的不是 `operator new` 分配的，这块内存可能是栈区、静态区或内存池，**不能随便释放**
2. 会导致**未定义行为**（比如释放栈内存会直接崩溃）

正确做法是：调用 `p->~A();` 手动析构，只清理对象内部资源，**不释放内存本身**（因为内存不属于你管理）。
