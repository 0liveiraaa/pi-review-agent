# Pi Review Agent — 认知框架

基于推理原子图与 Loop 工程的特化复习出题系统。本文件定义领域语言，不涉及实现细节。

## 核心实体

**推理原子 (Reasoning Atom)**：
解题过程中不可再分的推理操作，是出题的基本构建单元。从真题、LLM 世界知识或章节材料中逆向提取。
_Avoid_: 认知原子, 思维原子, 原子步骤

**原子图 (Atom Graph)**：
以推理原子为节点的无向带权图，边权为原子在题解中的共现强度。按 profile 独立维护。代码层在图上执行路径选择以确定出题方案。
_Avoid_: 认知关联图, 原子关联图, 原子网络

## 工作流编排

**技能回路 (Skill Loop)**：
将 agent 工作流封装为有向图的节点序列。每个节点内部为 ReAct 循环，满足出条件后流转至下一节点。节点可嵌套封装，多个 Skill Loop 可组合为多 agent 系统。技能回路采用混合主控：代码约束必须发生的状态转移和安全边界，agent 在节点内部完成语义判断、内容生成、解释和策略建议。
_Avoid_: Loop工程, harness, 工作流链

**节点 (Node)**：
技能回路中的一个工作阶段。五要素：目标 (goal)、上下文 (context)、可调用工具 (tools)、机制 (mechanism)、出入条件 (entry/exit conditions)。机制是满足可检验条件后自动执行的代码——上下文压缩、hook、cron 均属此类。
_Avoid_: 阶段, 步骤, 工位

**技能回路蓝图 (Skill Loop Blueprint)**：
由 skill 或 profile 声明的回路设计说明，描述场景目标、节点集合、允许的工具、转移规则、日志要求和优化准则。它指导 agent 使用工作流编排能力生成或调整技能回路，但不等同于某次运行实例。
_Avoid_: 提示词模板, 固定流程, 配置片段

**回路运行记录 (Loop Run Log)**：
技能回路执行时产生的结构化证据，记录节点进入/退出、工具调用、用户动作、agent 输出、失败原因和转移依据。回路运行记录用于会话后复盘和回路优化，不替代答题归档或学习画像。
_Avoid_: 普通日志, debug 日志, 聊天记录

**回路优化建议 (Loop Improvement Suggestion)**：
基于回路运行记录、运行数据和用户反馈生成的工作流调整建议。它可以建议调整节点顺序、节点条件、工具选择等几大要素，但必须经用户确认后才能成为长期配置。
_Avoid_: 自动改配置, 模型自我更新, 隐式学习

**回路配置叠加 (Loop Configuration Overlay)**：
技能回路配置的三层合并模型：系统默认回路提供稳定基线，Profile 资料包可声明可分享的场景推荐回路，用户私有运行数据可保存个人覆盖配置。运行时按默认回路、Profile 回路、用户覆盖配置的顺序叠加。
_Avoid_: 单一配置源, 用户数据混入资料包, 每次临时拼流程

## 知识点关联

**知识点 (Knowledge Point)**：
课程大纲定义的概念或定理，是复习的基本覆盖目标。在原子图中，知识点不作独立节点，而是作为标签挂载在推理原子上。
_Avoid_: 考点, 知识单元

**知识点标签 (KP Tag)**：
原子数据中记录的知识点引用字段。一个原子可携带多个标签。代码层依此过滤原子、计算路径跨域度。
_Avoid_: 知识点字段, KP关联

**原子-知识点映射 (Atom-KP Mapping)**：
推理原子与知识点的多对多标签关系。在原子提取阶段由 LLM 同步建立。一个知识点通常对应多个原子，一个原子可服务多个知识点。
_Avoid_: 知识点归属, KP归属

## 出题流程

**原子方案 (Atom Plan)**：
代码层根据目标知识点和难度在原子图上选择的一段原子组合（含边权重信息）。作为出题节点的主要上下文注入 LLM。由入口 hook 自动计算生成，非 agent 主动调用。
_Avoid_: 原子路径, 原子链, 原子序列, 原子子图

**难度分 (Difficulty Score)**：
0-100 的连续值，由原子方案的链长、边权稀度、跨域度加权计算。代码层选择原子方案时以目标难度分区间为约束。展示时映射为 S-R ~ C-A 标签供用户理解。
_Avoid_: 难度系数, 难度指标, 难度评级

## 数据构建

**种子池 (Seed Pool)**：
用于逆向提取推理原子的题解集合。真题题解为主源，LLM 世界知识为冷启动补充，章节材料中的例题为辅助。旧的 problem_templates 由原子方案替代，不再纳入种子池。
_Avoid_: 种子数据, 源数据, 训练数据

**数学结构 (Mathematical Structure)** ⚠️ 暂定：
具有内在生成力的数学配置（母题胚）。独立于原子图，在出题时由 LLM 决定是否选用。非题型（choice/judgment/short_answer 等），非旧版 problem_templates。方向：来源于构造直觉和逆向抽象，是可用于承载多种原子组合的可构造数学框架。具体定义和格式待后续确定。
_Avoid_: 题型, 题目模板, 出题格式

**Profile 资料包 (Profile Package)**：
一门课程或学习主题的可复习资料集合，包含学科说明、知识索引、卡片、章节材料、考点总结和质量说明。它是可迁移、可审查、可版本化、可分享的学习资产。
_Avoid_: 资料缓存, profile 文件夹, 课程包

**运行数据 (Runtime Data)**：
用户在复习过程中产生的私有状态，包括答题归档、总结、学习画像和进度信息。运行数据不属于可分享的 Profile 资料包。
_Avoid_: profile 数据, bundled 数据, 用户资料包

**演进回路 (Evolution Loop)**：
项目迭代的治理闭环：发现问题、修复技术债、验证行为、同步文档、再进入下一轮规划。它约束工程节奏，不等同于用户复习流程。
_Avoid_: 开发流程, 版本计划, 随手整理
