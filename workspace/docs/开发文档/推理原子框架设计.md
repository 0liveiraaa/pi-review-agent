## 1. 框架概览

### 1.1 三层实体

```
知识点 (Knowledge Points)   →  考什么：课程大纲的概念和定理
推理原子 (Reasoning Atoms)  →  怎么想：解题中的推理操作
数学结构 (Math Structures)  →  怎么装：具有生成力的数学配置 ⚠️ 暂定
```

三者独立定义、协同运作。知识点是标签（挂在原子上），原子是构建单元（带知识点标签，以无向图组织），数学结构是可选的出题框架（由 LLM 自由选用）。

### 1.2 出题流程（新旧对比）

```
旧流程 (prompt 驱动):
  选 KP → agent 参考 review-question skill → 自由生成题目
  问题: 题目质量不稳定，难度由 agent 主观判断

新流程 (原子图驱动):
  选 KP + 目标难度
    → 代码计算原子方案 (Atom Plan)
    → 注入出题节点上下文 (技能回路)
    → LLM 基于原子方案生成题目 → schema 校验 → 输出
  改进: 难度由图的拓扑特征保证，agent 只负责润色成完整题干
```

## 2. 推理原子与原子图

### 2.1 原子数据结构

```json
{
  "id": "atom_gcd_001",
  "name": "辗转相除求最大公因数",
  "description": "用带余除法反复替换被除数与除数，直到余数为零，最后非零余数即为 gcd",
  "knowledge_points": ["euclidean-algorithm", "gcd"],
  "embedding": [0.12, -0.34, ...],
  "source": {
    "type": "past_paper",
    "ref": "past_papers/questions/q_2022_final_003.md"
  }
}
```

### 2.2 原子图结构

- **类型**：无向带权图
- **节点**：推理原子
- **边**：原子在题解中的共现，权重 `ω_ij = log(1 + n_ij)`
- **作用**：按 profile 独立维护
- **构建**：从种子池提取原子 → 向量聚类去重 → 统计共现建边 → 剪枝过泛节点

### 2.3 种子池

优先级顺序：
1. **真题题解**（主源）—— 从 `past_papers/questions/` 逆向提取
2. **LLM 世界知识**（冷启动补充）—— 图稀疏时直接让 LLM 生成原子
3. **章节材料例题**（辅助）—— 从 `chapters/` 提取

旧 `problem_templates/` 方案废弃，由原子方案替代。

### 2.4 冷启动策略

1. 手工准备 lisan 第 1 章的离散数学真题 2-3 道（题目 + 详细解答）
2. 从真题中逆向提取第一批原子（来源 2）
3. LLM 以世界知识扩充原子（来源 4），达到可用密度
4. 后续真题层完善后，真题反推为主，LLM 知识为辅

## 3. 原子方案选择

### 3.1 概述

原子方案由代码层（非 agent）计算。agent 进入出题节点时，入口 hook 已经选好方案并注入上下文。

### 3.2 输入

- 目标知识点（KP ID，用户选择或推荐结果）
- 目标难度分区间（开局配置）

### 3.3 算法（首版：约束随机游走 + 过滤）

```
1. 定位入口：从携带目标 KP 标签的原子集合中随机选取起始节点
2. 随机游走：度惩罚游走 n 步（p ∝ ω / deg^α），产生候选路径
3. 过滤：只保留难度分落在目标区间的候选
4. 多样性：多次采样，返回 N 条不同的候选方案
5. 选择：agent 从候选中选择最合适的（或代码随机选一条）
```

### 3.4 超参数

| 参数 | 含义 | 首版默认值 |
|------|------|-----------|
| α | 度惩罚强度 | 待校准 |
| K | 目标原子数（与难度关联） | 1-10 |
| N | 候选方案数 | 3 |

### 3.5 难度计算公式

```
difficulty = α · chain_length × (1 + β · sparsity) × (1 + γ · cross_domain)

其中:
  chain_length  = len(atom_plan)           // 原子数量
  sparsity      = 1 - mean(edge_weights)   // 低关联度原子占比
  cross_domain  = KP标签集的Jaccard距离    // 跨知识点群的程度

系数 α/β/γ：待根据实际出题数据校准
```

### 3.6 难度分 → 展示标签映射

| 连续分区间 | 展示标签 | 典型特征 |
|-----------|---------|---------|
| 0–20 | S-R | 1 个原子，单点回忆 |
| 20–40 | S-U | 1-2 个高关联原子 |
| 40–60 | M-U | 2-3 个直接边连接的原子 |
| 60–80 | M-A | 3-5 个原子连通路径 |
| 80–100 | C-A | 5+ 原子，跨 KP 域，含低权桥接边 |

## 4. 技能回路：出题节点

### 4.1 节点设计

```
节点: generate_question
  ┌─────────────────────────────────────────────┐
  │ context:                                    │
  │   - 当前 KP（用户选择/推荐结果）              │
  │   - 目标难度分区间                          │
  │   - 原子方案（由入口 hook 计算）              │
  │   - 可选数学结构池                          │
  │   - profile 资料摘要 (subject.md, cards)     │
  │                                             │
  │ tools:                                      │
  │   - read cards / chapters / exam_points     │
  │   - review_answer (产题)                    │
  │                                             │
  │ entry_condition:                            │
  │   - 已确定目标 KP                           │
  │   - 已确定目标难度分区间                      │
  │                                             │
  │ exit_condition:                             │
  │   - 产出合法 Question JSON                   │
  │   - 通过 typebox schema 校验                 │
  │   - 原子覆盖度满足要求                       │
  │                                             │
  │ internal ReAct:                             │
  │   - agent 理解原子方案语义                   │
  │   - agent 可选读资料补充上下文               │
  │   - agent 可选套用数学结构                   │
  │   - agent 生成题目 → 校验 → 不通过则修正     │
  │                                             │
  │ hooks:                                      │
  │   - on_entry: 计算原子方案 (select_atom_plan)│
  │   - on_exit:  记录 selected_atoms           │
  └─────────────────────────────────────────────┘
```

### 4.2 原子方案注入 prompt 的格式

代码层选出的原子方案在注入 prompt 时，呈现为结构化的推理步骤序列：

```
[Atom Plan]
1. 素因数分解 (KP: prime-factorization, gcd)
   → 将给定整数分解为素数幂的乘积
2. 辗转相除求 gcd (KP: euclidean-algorithm, gcd)
   → 用带余除法反复替换被除数与除数
3. Bezout系数回代 (KP: bezout-identity, linear-diophantine)
   → 从辗转相除过程回代展开，求线性组合系数
4. 通解形式推导 (KP: linear-diophantine, modular-arithmetic)
   → 从特解推导 ax+by=c 的全部整数解

Difficulty: 72 (M-A)
Required KP coverage: bezout-identity, linear-diophantine
```

agent 的职责：基于这组推理步骤，构造一道完整题目，使解题过程自然经过这些原子。可选：套用一个数学结构（如「存在性+构造型」「优化+约束型」）。

## 5. 与现有系统的集成路径

### 5.1 新增模块

```
workspace/lib/
  atom_graph.mjs        ← 原子图构建与维护
  atom_plan.mjs         ← 原子方案选择引擎
  atom_extraction.mjs   ← 种子池管理与原子提取协议

workspace/state/
  atom_graphs/
    {subject_id}.json   ← profile 级原子图持久化
```

### 5.2 修改模块

- `review_engine.mjs`: `buildReviewStartPrompt()` 改为注入原子方案
- `review-question` skill: 从「自由出题规则」改为「原子方案→题目生成规则」
- `review-core` skill: 增加原子方案相关工具契约

### 5.3 废弃内容

- `problem_templates/` 旧模板体系

### 5.4 兼容期

冷启动阶段原子图未达可用密度时，出题引擎回退到旧 prompt 路径。原子图密度达到阈值后自动切换。

## 6. 实施路线

### 阶段 1：lisan 冷启动（主线 B 路径）

1. 手工准备 2-3 道离散数学第 1 章真题（题目+详细解）
2. 实现原子提取协议（LLM 逆向提取 + embedding 聚类）
3. 从真题 + LLM 世界知识构建第一批原子，建图
4. 实现简单约束随机游走 → 原子方案选择
5. 实现出题节点（入口 hook + 上下文注入）
6. 一次真实复习 session 验证

### 阶段 2：难度公式校准

1. 收集阶段 1 产出的原子方案 → 题目对
2. 人工标注实际难度
3. 回归校准 α/β/γ 系数

### 阶段 3：真题层完善

1. 建 `past_papers/` 目录结构（主线 C1）
2. 真题拆解和原子反推（主线 C3）
3. 真题反推为主源，减少 LLM 知识占比

### 阶段 4：数学结构沉淀

待数学结构的定义和格式明确后，从真题和原子方案中逆向抽象。
