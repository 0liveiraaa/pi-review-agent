# pi-review 待排查问题

记录于 2026-06-08，复习会话中发现。

## 1. review_chapter 章节卡片未渲染

- **现象**：章节学习模式下调用 `review_chapter`，仅返回 `action: "practice"`，TUI 中没有展示章节内容卡片
- **预期**：应渲染包含章节标题、知识点列表、节列表的内容卡片
- **排查方向**：`review_chapter` 工具实现中 TUI 组件（ChapterCard/ChapterPanel）是否被正确挂载和显示

## 2. TUI 渲染问题

### 2a. 长行左侧截断

- **现象**：卡片内容一行过长时，最左边的文字不显示
- **排查方向**：Viewport/Paragraph 组件的 wrap/scroll 属性配置，水平溢出处理

### 2b. 滚动位置回弹

- **现象**：看完一段内容后，滚动位置自动跳回最上层，需手动拉下来
- **排查方向**：状态更新触发 re-render 时 ScrollView/List 的 scroll_offset 被重置

## 3. revision draft 命名层层嵌套

- **现象**：多次修订后 subjectId 变成 `cpp-oop__draft_20260608__draft_20260608`
- **预期**：应回溯到原始 subjectId 再生成，如 `cpp-oop__draft_20260608_v2`
- **排查方向**：`review-fix` 创建 revision draft 时的命名逻辑，应检查 `revisionOf` 链

## 4. 草稿目录累积

- **现象**：每次修订启用后，旧版本归档但目录保留，多次修订后 `review_profiles/` 下目录堆积
- **建议**：归档时考虑清理或提供 `--prune` 选项

## 5. review_answer 长题目文本被截断/省略

- **现象**：2026-06-09 章节学习模式（chapter_study），调用 `review_answer` 展示含代码片段的简答题时，TUI 中题目文本被省略，输出显示「代码被省略了TUI问题记录一下… 已省略 14 行」，导致用户无法看到完整题目内容
- **预期**：题目文本（含代码块）应完整展示给用户，支持滚动查看
- **影响**：用户因看不到完整题目而无法作答，只能退出答题
- **排查方向**：
  - `review_answer` 工具内部 TUI 渲染组件（如 Paragraph/MarkdownView）的内容长度限制
  - 输出日志的截断阈值（50KB/2000行限制）与 TUI 展示的截断策略是否一致
  - pi-agent 上下文压缩机制是否误将题目文本当作冗余历史而压缩
  - 建议在工具层面增加「展开全文」交互或支持分页显示长题目
