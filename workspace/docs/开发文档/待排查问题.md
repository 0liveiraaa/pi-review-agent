## 3. review_answer 长题目省略

- 现象：含代码块的长简答题被显示为“已省略 N 行”，用户无法看到完整题干。
- 状态：已修复，题干先进入可滚动题目面板，再进入选择/输入控件。
- 复验路径：构造含 50 行代码块的简答题，确认 J/K 可滚动阅读，Enter 后可继续作答。

## 4. 题后菜单冗余

- 现象：题后菜单出现“提示/追问/提高难度”等时机不合适的选项。

  - 下一题
  - 看卡片
  - 看章节
  - 总结
  - 退出
- 说明：提示/追问放在答题过程中；难度选择前置到 `/review` 开局配置。

## 5. 学习画像阻塞启动流程

- 现象：每次选择 profile 后弹出一整屏学习画像，并要求 Enter 继续。
- 状态：已修复，改为非阻塞通知/提示，不再打断 `/review` 启动流程。
- 关联变化：学习画像现在优先从资料包私有 `_user/learning_profile.json` 读取。

## 6. revision draft 命名层层嵌套

- 现象：多次修订后 subjectId 变成 `cpp-oop__draft_20260608__draft_20260608`。
- 状态：已修复。
- 当前规则：
  - 首次修订：`{root}__draft_{YYYYMMDD}`
  - 当天再次修订：`{root}__draft_{YYYYMMDD}_v2`
  - 继续递增：`_v3`、`_v4`
- 说明：命名会沿 `revisionOf` / `revisionRoot` 回溯到原始 subjectId，并兜底剥离已有 draft 后缀。

## 7. 草稿和归档目录累积

- 现象：每次修订启用后，旧 active 会归档但目录保留，`review_profiles/` 下目录逐渐增多。
- 状态：保留但隐藏。
- 当前策略：
  - `/review` 只显示 active。
  - `/review-fix` 只显示 active 和 draft。
  - archived 不进入普通使用流。
- 后续规划：增加 `/review-doctor` 或 `review_profile_prune`，默认 dry-run，用户确认后再删除。

## 8. summary / learning profile 全局化

- 现象：旧实现把 summaries 和学习画像放在全局 `archive/`、`state/learning_profiles/`，多资料包使用时容易混杂。
- 状态：已修复。
- 当前规则：
  - 新 summary 写入 `review_profiles/{subjectId}/_user/summaries/`。
  - 新学习画像写入 `review_profiles/{subjectId}/_user/learning_profile.json`。
  - 旧全局 `state/learning_profiles/{subjectId}.json` 保留 fallback。
- 说明：`_user/` 是用户私有运行数据，不属于可分享资料包内容。
