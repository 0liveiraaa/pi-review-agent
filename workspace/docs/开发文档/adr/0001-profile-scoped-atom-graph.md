# 原子图按 profile 独立维护

原子图不设全局共享层，每个 profile 独立构建和维护自己的推理原子图。后续可演进为分层架构（profile 本地图 + 全局通用原子引用），但首版不实现。

## 考虑过的方案

**方案 A：全局共享图。** 所有 profile 的原子纳入同一张图，跨科原子可互连。
- 优点：跨域迁移、无重复提取
- 缺点：冷启动复杂度高，离散数学的原子和 C++ 的原子在图中共存可能导致不合理的跨域路径

**方案 B：profile 独立图（选用）。** 每个 profile 带自己的原子图。
- 优点：冷启动简单，无跨科噪声，出题范围天然约束在 profile 知识域内
- 缺点：通用原子（如「反证法」「构造反例」）会被重复提取

**方案 C：分层图。** profile 本地图可引用全局共享的通用原子。
- 后续路线，首版不实现。

## 为何选 B

当前首版以 lisan（离散数学）为唯一验证 profile。B 方案最快达到可用密度，且不需要处理跨域原子合并的语义去重问题。待多 profile 场景成熟后，迁移到 C（分层）的路径清晰：只需增加全局通用原子层 + 引用机制。
