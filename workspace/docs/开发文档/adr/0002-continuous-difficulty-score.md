# 难度采用连续分计算

出题难度从旧的 S-R ~ C-A 离散标签改为 0-100 连续分，由原子方案的图拓扑特征（链长、边权稀度、跨域度）加权计算。展示层保留 S-R ~ C-A 标签作为用户可读映射。

## 考虑过的方案

**方案 A：保留旧 5 级离散标签。** S-R / S-U / M-U / M-A / C-A 由 agent 主观判断。
- 优点：简单，现有代码不改
- 缺点：粒度粗，agent 对「M-A 难度」的理解不一致，无法做细粒度难度控制

**方案 B：连续分（选用）。** 由代码层计算，公式：`difficulty = α·链长 × (1 + β·稀度) × (1 + γ·跨域度)`。
- 优点：可算且可复现，难度由所选原子方案的拓扑特征保证而非 agent 猜测；可以精确映射到展示标签
- 缺点：需要校准参数 α/β/γ

## 为何选 B

旧体系的核心问题是「难度」依赖 agent 主观判断——agent 说不清 M-A 和 C-A 的区别，题目难度实际上由 prompt 修辞水平决定。连续分把难度推到代码层：选出什么样的原子方案，就已经决定了难度区间。agent 只负责润色成题干。
